ChatGTM automations → cursor.omarsimon.com

Four scheduled ChatGTM agents call the Next.js API on Vercel. All persistent state lives in Neon Postgres. Omar works cold sequences and intent outreach in the /admin dashboard (same database, no separate ChatGTM write path for UI edits).

GET (read) POST (create / upsert) PATCH (update state) Gmail (external)

Data split

  • prospects — cold outbound + demo pages (/p/slug)
  • outreach_* — intent / warm-signal contacts only
  • Intent demos create a shadow prospects row (source=outreach) hidden from Sequences until promoted

Typical daily order

  • Reply Checker — stop sequences for anyone who replied
  • Intent Signal — ingest yesterday’s warm contacts
  • Outreach Orchestrator — send due cold emails + flagged intent emails
  • Prospecting Blitz — add new cold prospects (on its own schedule)

Auth

All ChatGTM HTTP steps use Authorization: Bearer CHATGTM_API_TOKEN. The admin dashboard uses a separate admin password session; both hit the same Neon database through the Vercel app.
